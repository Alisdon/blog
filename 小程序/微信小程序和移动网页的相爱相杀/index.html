微信小程序和移动网页的相爱相杀

2020-03-18 小程序 小程序 0 0 Comments Word Count: 400(words) Read Count: 1(minutes)

两者身为同一个时代的产物，存在太多志同道合、“异性相吸”的魔力，又有着不拘一格、“同性相斥”的魅力。但可以确定的是，两者身上都有一种为了实现世界统一的魄力。

生死之战

pk 项 h5 mp-weixin
运行环境 浏览器 微信
系统API权限 弱 强
分享朋友圈 能 不能
扫码识别 能 能
支付能力 多种 微信支付
使用流畅/平滑 一般 良好
入口数量 少 多
专属入口 无 有
简易功能成本 低 低
系统功能成本 高 低
迭代周期 短 较长
外部限制 较少 多

跨界合作

准备工作

  1. 注册小程序账号必须为企业的，个人类型的小程序暂不支持使用
  2. 微信 6.7.2 版本以上(目前安卓最新版 7.0.12/苹果 7.0.11)
  3. 需登录小程序管理后台配置业务域名
  4. 基础库1.6.4以上

命令部署

  1. 业务域名中配置的就是小程序以及 H5 和 H5 中引用 iframe 的域名
  2. 将校验文件放置在将要嵌套的业务域名的根目录

统一战线

  1. 嵌入的 h5 不能调用支付接口
  2. 嵌入的 h5 拿不到分享成功失败的信息

互帮互助

  1. mp -> h5 使用 src 地址传值
  2. h5 -> mp postMessage, web-view 绑定 bindmessage 进行接收
  3. weixinjssdk

tips: bindMessage为网页向小程序 postMessage 时，会在特定时机（小程序后退、组件销毁、分享）触发并收到消息
